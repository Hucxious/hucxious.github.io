Transformer架构凭借其Attention机制，训练速度快且推理能力强，在Backbone上一众成为基础架构，收到各大模型青睐。但由于其复杂度，随着推理进行，KV cache存储越来越多，其速度也会越来越慢。Structured SSM虽然复杂度仅为，可以等价为RNN和CNN，但由于其通过序列建模压缩了上下文，其推理效果不如Transformer。Mamba则在二者基础上，实现了一个训练和推理速度均较快且效果丝毫不逊于Transformer的架构，其复杂度仅为。下面结合几篇相关论文展开介绍和理解。

Mamba的创新点

这篇文章中指出，传统的LTI SSM由于其时不变特性，导致状态更新固定，其中作者特意举出了两个例子：Selective Copy 和 Induction Heads。前者说明，传统的卷积和LTI SSM缺乏选择性，后者说明其缺乏了泛化能力。因此，Mamba针对这些问题提出了创新，具体包括以下几个方面：

  1. Selective State Space Model：在传统地LTI 状态空间模型基础上，引入了输入依赖（input-dependent）的状态空间模型。用参数将连续时间进行离散化，分别是输入和输出的投影，从而实现了一个可变时间尺度。

  2. Selection Mechanism：利用Selection机制来代替Attention机制，作为一种隐式记忆更新。其中在里面发挥了类似门控的作用，越小，则忽略，保持记忆；越大，则重置记忆。

  3. Hardware-aware Selective Scan：这里更是集中在对GPU这种底层的处理，解决了参数量巨大占用内存的问题，主要方案是核融合（Kernel Fusion）、并行扫描（Parallel Scan）和重计算（Recomputation）。

Vision Mamba: Efficient Visual Representation Learning with Bidirectional State Space Model

Vision Mamba（Vim）是在Mamba出来后比较早的一批将序列模型引入到2维图像中的一篇文章。Attention的复杂度呈现时间的二次方，故对于高分辨率的图像做推理耗时长、占用内存大。由于原始的Mamba是单向且无位置感知的，无法直接应用在视觉图像上，故作者针对该两个问题进行了改进，使得Mamba成功应用在视觉图像上。SSM在Vim中的地位，就等效于self-attention在Transformer的地位，是整个模型的核心。

模型创新点

  1. 基于Mamba，提出了一种通用的视觉主干（Backbone）——Vim Block，引入了双向SSM（Bidirectional SSM）。

  2. 针对Mamba无位置感知的局限性加入了Position Embedding。对应论文中的公式：

VMamba: Visual State Space Model

VMamba和Vim都是比较早期对Mamba在视觉上应用的模型。与Vim不同，VMamba主要集中在对多方向扫描的和层次化网络结构的融合（Vision Mamba说的~）。

Mamba YOLO: A Simple Baseline for Object Detection with State Space Model

Mamba YOLO这篇文章，说实话我感觉他的整个模型已经脱离了YOLO的原先架构了，在YOLO基础上保留了P3/P4/P5多尺度输出以及稠密层进行训练。作者参考了当前的一些主流模型如DETR，在Backbone用SSM（Mamba的核心）代替了CNN，在Neck融合单元用ODSSBlock代替C2f，全局感知上用的是序列建模也并非局部感受野堆积。总的来说，是将Mamba的思想融合进了YOLO之中。

模型创新点

  1. 在YOLO中加入复杂度为的SSM模型，保留了YOLO无需进行预训练和大规模数据集的性质。

  2. 提出了ODSSBlock的新模块，补偿了本地对SSM模型的兼容性。该模块由SS2D和RGBlock构成，前者相当于引入Scan机制，后者即引入了门控机制。

Mamba YOLO模型

Mamba YOLO的主要模型架构如下图所示，可以看到整个模型保留了YOLO的整体形式，在主Backbone将改进的Mamba块代替CNN，在Neck采用PAFPN，Head为一个解耦的头进行输出，仍然保留了YOLO的P3/P4/P5多尺度输出。

PAFPN：特征金字塔（FPN）是识别系统中用于检测不同尺度目标的基本组件。PAFPN在FPN的自上向下的路径之后添加了一个自底向上的路劲，YOLOv4中已经这么干了。但是作者在这里，对原来的PAFPN中的C2f也都替换成了ODSSBlock，如作者所言：

“In the neck, we follow the design of PAFPN (Jocher, Chaurasia, and Qiu 2023) using the ODSSBlock module instead of C2f to capture a more gradient-rich information flow.”

SS2D：原本的SSM是一个一维时序模型，按时序进行扫描即可；但由于对视觉图像进行处理，因此要改成二维扫描方式。SS2D的核心就是对图像依次从四个方向进行了扫描，将扫描得到的“离散序列”送入S6模型（Mamba核心），再通过扫描融合（Scan Merge）将序列返回成二维图像。这一步的作用在于获得全局空间感知能力。

RGBlock：RGBlock借鉴了Gated MLP的思想，同时引入DW-Conv和Residual，代替了MLP。DW-Conv是深度可分离卷积（Depthwise separable convolution），相比常规的卷积操作，其参数数量和运算成本比较低。这里使用DW-Conv而不是Conv的原因，可能是为了单纯实现空间上的补偿同时不干扰SSM的递归结构，不混通道。作者也给出了最终的输出特征公式：

ByteTrack: Multi-Object Tracking by Associating Every Detection Box

这是一篇视觉追踪的文章，主要介绍了对多目标的追踪的优化，提出了一个BYTE连接的办法。因为前面看到了YOLO+ByteTrack成为了目前的主流的一种目标跟踪算法，就想着能不能在这上面找找创新，就把原文拿来读了读。

核心观点

ByteTrack核心强调点就是：Low confidence detection boxes sometimes indicate the existence of objects, e.g. the occluded objects. 作者强调，现在我们已经有了较好的检测头（detector），比如YOLOX、CenterNert等；也有了很好的相似度判断方法（similarity），比如交并比（IoU）、外观相似（Re-ID）、匈牙利算法等。但现在我们的常用链接方法（association）就是只用高分的检测结果，而忽略掉了低分的检测结果，而这种策略导致了我们对遮挡场景判断成功降低，以及预测轨迹出现断裂。低分的检测结果中往往隐含了动态信息，于是作者提出了一个两阶段匹配方式，从而实现了对高分和低分探测的综合应用，提高检测效率。

High-score Detections —> Similarity#1

Low-score Detections —> Similarity#2

其中 Similarity #1 就是标准 的MOT 的关联方式，把高得分的detection匹配给tracklet，具体方式可以有IoU和Re-ID。而 Similarity #2 则是用低分的detection，将第一阶段为匹配的tracklets进行匹配，而且具体方式只用 IoU，不用 Re-ID（因为这个时候探测得分本来就低，外观相似性并没有很值得参考）。
